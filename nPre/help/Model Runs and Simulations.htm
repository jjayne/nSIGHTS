Model Runs and Simulations

Each time the well test analysis model is executed within nSIGHTS, a model run is created. To execute the well test analysis model (and create a model run), a command in the Run menu is selected (see Running Simulations).

Depending on the model options selected, a model run within nSIGHTS may consist of one or many simulations or cases. A simulation is the calculation of one flow solution, with one set of parameter values. A case is one or a group of simulations depending on the simulation mode:

  • In forward mode, a case is one simulation, with one combination of suite, range or sampled parameters.

  • In optimization mode, a case is a group of simulations, based on one set of non-fitting parameters, such as a set of sampled parameters or one combination of suite or range variables. Each simulation of a optimization mode case has adjusted values of fitting parameters, with the goal of providing an optimal fit between simulated results and field data.

Simulation mode indicates general model function. For example, optimization mode indicates that the model will conduct several simulations, adjusting the values of user-specified parameters for each simulation, to obtain an optimal fit between simulated and field data. Each model run has only one simulation mode. The simulation modes available are identified and described in Configuration Main Tab.
